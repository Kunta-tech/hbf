# BFO Naming Convention

To prevent naming collisions between user-defined `cell` variables and compiler-generated temporaries, the HBF compiler follows a strict naming convention for all BFO (Brainfuck Object) output.

## User Variables
Variable names defined in HBF source code are preserved as-is in BFO if they are `cell` variables.
- Example: `cell x = 10;` -> `set x 10`

## Compiler-Generated Names
All names generated by the compiler are prefixed with `__hbf_` to ensure global uniqueness.

### 1. Temporaries
Used for complex expression materialization and printing:
- Name: `__hbf_tmp`

### 2. Loop Counters
Used for anonymous `forn` loops:
- Pattern: `__hbf_forn_{ID}`
- Example: `__hbf_forn_0`, `__hbf_forn_1`

### 3. Physical Array Cells
Underlying cell names for physical arrays (`cell[]`):
- Pattern: `__hbf_cell_{ARRAY_NAME}_{INDEX}`
- Example: `cell arr[2];` -> `__hbf_cell_arr_0`, `__hbf_cell_arr_1`

## Reserved Prefix
Users are advised to avoid naming their `cell` variables with the `__hbf_` prefix to prevent potential conflicts with future compiler features.
